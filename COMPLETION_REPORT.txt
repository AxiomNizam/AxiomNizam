USAGE PATTERNS
==============

Pattern 1: Admission Control
```go
admCtrl := NewAdmissionController(eventBus, policy, rbac, quota)
resp, _ := admCtrl.Admit(ctx, request)
if !resp.Allowed { return fmt.Errorf("Denied") }
```

Pattern 2: RBAC Authorization
```go
rbac := NewRBACEngine()
allowed, _ := rbac.CanPerform(ctx, user, resource, verb, ns)
if !allowed { return fmt.Errorf("Forbidden") }
```

Pattern 3: Event Subscription
```go
eventBus.SubscribeToResourceEvents(types, handler)
eventBus.PublishResourceEvent(ctx, event)
```

Pattern 4: Controller Reconciliation
```go
reconciler := NewControllerReconciler(...)
duration, err := reconciler.Reconcile(ctx, key)
```

Pattern 5: Resource Validation
```go
validator := NewAPIResourceValidator(schema)

```

PERFORMANCE CHARACTERISTICS
============================

Admission Controller:
- Per-phase execution: <1ms typically
- Parallel webhook dispatch: O(n) where n = webhook count
- Early exit on denial
- Average total time: 5-10ms

RBAC Engine:
- Permission check: O(1) lookup + O(n) rule matching
- Audit logging: O(1) ring buffer append
- Memory: Fixed size (10K audit entries)

Event Bus:
- Publish: O(1) channel send
- Subscribe: O(1) list append
- Watch/filter: O(n) where n = subscriber count
- Aggregation: Reduces memory footprint

Controller Reconciliation:
- Phase execution: Typically 10-100ms per phase
- Observer execution: Configurable intervals
- Retry backoff: 1s → 2s → 4s → 8s → 16s (max)
