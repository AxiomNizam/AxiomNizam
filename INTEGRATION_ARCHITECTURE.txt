## AxiomNizam Platform - Complete Integration Architecture

### System Overview

AxiomNizam is now a complete enterprise-grade data platform with integrated:
- **CLI Interface**: Kubernetes-style command structure
- **API Banks**: API discovery and catalog management  
- **Data Mesh**: Domain-driven data product topology
- **Compliance**: Comprehensive audit and access control
- **Monitoring**: Health, alerts, metrics, quality tracking
- **Observability**: Events, audit logs, lineage tracing

---

## Architecture Layers

### Layer 1: Core Data Models

**Data Mesh** (`internal/mesh/datamesh.go`)
```
DataMesh
├── DataDomain (Finance, Operations, etc.)
│   ├── Name, Owner, Namespace
│   ├── Tags
│   └── DataProducts
│       ├── Name, Owner, Schema
│       ├── SLA (Availability, Latency)
│       ├── Ports (API access points)
│       └── Subscriptions
└── LineageTracer (upstream/downstream analysis)
```

**API Banks** (`internal/apibanks/manager.go`)
```
APIBank
├── Name, Owner, Namespace
├── Tags
├── APIs[]
│   ├── Name, Endpoint
│   ├── Kind, DataClasses
│   ├── Owner, Description
│   └── SLA
└── APIBankCatalog (search, discovery)
```

### Layer 2: Integration Systems

**Data Platform Integration** (`internal/integration/data_platform.go`)
- CatalogIntegration: Unified search across APIs and data products
- DataQualityMonitor: Quality scoring for data products
- DataLineageAnalyzer: Data flow tracing and impact analysis
- APIBankObserver: Event handling for bank changes
- DataMeshObserver: Event handling for mesh changes

**Compliance & Security** (`internal/integration/compliance.go`)
- ComplianceAuditor: Audit trail recording and reporting
- DataAccessControl: Access control enforcement
- Operation auditing with policy evaluation
- AuditFilter: Query and filter audit records

**Monitoring & Health** (`internal/integration/monitoring.go`)
- HealthMonitor: Component health tracking
- PlatformMetricsCollector: Metrics aggregation
- AlertManager: Alert generation and management
- ComponentHealth: Per-component health status

### Layer 3: CLI Commands

**Integration Commands** (`cmd/axiomnizamctl/integration.go`)

```
Health Management
├── axiomnizamctl health check
└── axiomnizamctl alerts [check|list]

Metrics & Observability
├── axiomnizamctl metrics collect
├── axiomnizamctl catalog [search|list]
├── axiomnizamctl compliance [report|audit]
├── axiomnizamctl quality check
└── axiomnizamctl lineage trace
```

## Data Flow Diagrams

### Create Data Product Flow
```
CLI (mesh product create)
  ↓
ComplianceAuditor (record operation)
  ↓
DataMesh (create product)
  ↓
APIBankObserver (trigger event)
  ↓
EventRecorder (record event)
  ↓
Metrics (update stats)
  ↓
ComplianceReport (include in audit)
```

### Search & Discovery Flow
```
CLI (catalog search --tag)
  ↓
CatalogIntegration (unified search)
  ├→ APIBankCatalog (search by tag/class/owner)
  ├→ DataMeshDiscovery (search data products)
  └→ Results aggregation
  ↓
Formatted CLI output
```

### Compliance & Access Control
```
Operation Request (data access)
  ↓
DataAccessControl (check authorization)
  ↓
ComplianceAuditor (record operation)
  ↓
PolicyEvaluation (evaluate policies)
  ↓
Decision (allow/deny)
  ↓
ComplianceReport (audit trail)
```

### Quality & Lineage Analysis
```
Quality Check Request
  ↓
DataQualityMonitor
  ├→ Check SLA presence
  ├→ Check schema definition
  ├→ Check port configuration
  └→ Calculate quality score
  ↓
LineageAnalyzer
  ├→ Trace downstream subscribers
  ├→ Trace upstream sources
  └→ Identify related products
```

## Integration Points

### 1. Event Recording
Every operation triggers:
- Compliance audit entry
- Event recording
- Metrics update

```go
GlobalComplianceAuditor.RecordOperation(ctx, operation)
GlobalEventRecorder.Event(ctx, reference, reason, message)
metrics.RecordEvent(duration)
```

### 2. Discovery & Search
Unified catalog provides single search interface:
- Search by tag, owner, data class
- Cross-platform results (APIs + data products)
- Consistent discovery experience

```go
results := GlobalCatalogIntegration.UnifiedSearch("financial")
// Returns APIs and data products matching tag
```

### 3. Health & Monitoring
Continuous system monitoring:
- Component health tracking
- Alert generation
- Metrics collection
- Status reporting

```go
health := GlobalHealthMonitor.CheckHealth(ctx)
metrics := GlobalPlatformMetricsCollector.CollectMetrics(ctx)
alerts := GlobalAlertManager.GenerateAlerts(ctx)
```

### 4. Quality Assurance
Data product quality tracking:
- Quality scoring (0-100%)
- SLA compliance
- Schema validation
- Subscription tracking

```go
report := GlobalDataQualityMonitor.GetQualityReport("Finance")
// Returns: average score, products, details
```

### 5. Access Control
Policy-driven access management:
- User authorization checks
- Operation recording
- Compliance reporting
- Access audit trail

```go
allowed := GlobalDataAccessControl.CanAccessProduct(ctx, user, domain, product)
GlobalDataAccessControl.RecordDataAccess(ctx, user, domain, product)
```

## Example Workflows

### Workflow 1: Create Data Product
```
1. axiomnizamctl mesh domain create --name Finance --owner team1
2. axiomnizamctl mesh product create --domain Finance --name Transactions
3. ComplianceAuditor records: "DomainCreation", "ProductCreation"
4. HealthMonitor reports: new domain added
5. QualityMonitor calculates: initial quality score
6. Metrics updated: domain count, product count
```

### Workflow 2: Subscribe to Data Product
```
1. axiomnizamctl mesh subscribe --domain Finance --product Transactions --subscriber team2
2. ComplianceAuditor records: "SubscriptionCreation"
3. DataMesh updates: subscription list
4. EventRecorder: "SubscriptionCreated" event
5. LineageTracer updated: new downstream consumer
6. Metrics updated: subscription count
```

### Workflow 3: Generate Compliance Report
```
1. axiomnizamctl compliance report
2. ComplianceAuditor aggregates audit entries
3. Policy evaluation applied
4. Risk assessment calculated
5. Operations summarized by type/user
6. Report with summary statistics
```

### Workflow 4: Monitor Data Quality
```
1. axiomnizamctl quality check --domain Finance
2. DataQualityMonitor iterates products
3. Scores each product: SLA (25%), schema (25%), ports (25%), owner (25%)
4. Calculates domain average
5. Reports detailed quality metrics
```

## System Architecture Principles

1. **Single Responsibility**: Each component handles one concern
2. **Loose Coupling**: Via integration layer and interfaces
3. **High Cohesion**: Related functionality grouped together
4. **Observable**: Comprehensive audit and event trails
5. **Auditable**: Every operation recorded and traceable
6. **Resilient**: Health checks and monitoring
7. **Discoverable**: Unified catalog and search
8. **Compliant**: Policy evaluation and access control
9. **Extensible**: Global managers support injection
10. **Testable**: Comprehensive test coverage
